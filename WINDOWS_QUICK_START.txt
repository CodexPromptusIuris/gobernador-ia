╔═══════════════════════════════════════════════════════════════════════════╗
║                    TROUBLESHOOTING                                         ║
╚═══════════════════════════════════════════════════════════════════════════╝

❌ "Docker not found"
   Solution: Install Docker Desktop
   Download: https://www.docker.com/products/docker-desktop

❌ "Port already in use"
   Solution: Stop other applications using port 5173
   Or restart your computer

❌ Batch file won't run
   Solution: Right-click run.bat > Run as Administrator
   Or open Command Prompt and run: docker compose up -d --build

❌ Takes too long
   First run downloads ~500MB images (normal)
   Next runs are much faster (30 seconds)

❌ Services won't start
   Solution: 
   1. Open Command Prompt
   2. Go to project folder: cd C:\path\to\project
   3. Run: docker compose logs
   4. Look for error messages

❌ Can't reach http://localhost:5173
   Solution:
   1. Wait 60 seconds (services starting)
   2. Run: docker compose ps
   3. Check all containers show "Up"
   4. Try: http://localhost:8080/health (backend test)

STILL NOT WORKING?

Open Command Prompt (Windows key + R, type: cmd, press Enter)

Then run:
  docker --version
  
If it says "Docker not found":
  → Install Docker from https://www.docker.com/products/docker-desktop

If Docker is installed:
  cd C:\path\to\your\project
  docker compose up
  
This will show you what's happening.
